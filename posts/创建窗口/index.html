创建窗口

具体环境配置：https://learnopengl-cn.github.io/01%20Getting%20started/02%20Creating%20a%20window/

GLFW

作用：创建窗口与OpenGL上下文

  • 为何要自己做？
    OpenGL 只定义了渲染管线的标准（即「你怎么画」），并未规定如何创建窗口或上下文（即「在哪里画、怎样拿到上下文」）。不同操作系统对窗口管理和上下文创建的接口各不相同：Windows 用 Win32 API，Linux 用 X11/Wayland，macOS 用 Cocoa

  • 抽象的好处与成本
    OpenGL 故意将这些与渲染无关的部分抽象出去，让标准更聚焦。但这也意味着开发者要自己写跨平台的窗口和上下文代码，或引入第三方库来统一接口（GLFW为其中之一）

  • 典型流程

    1. 初始化库（如 glfwInit()​）
    2. 创建窗口并指定 OpenGL 版本／Profile
    3. 切换到该上下文（glfwMakeContextCurrent​）
    4. 进入渲染循环：处理事件→绘制→交换缓冲

GLAD

作用：运行时加载 OpenGL 函数指针

  • 为什么要加载？
    OpenGL 的实现（驱动）由各显卡厂商提供，不同版本和扩展中的函数并不会统一导出到系统的链接库里。编译阶段无法确定它们的地址，必须在运行时询问驱动：“请给我 glGenBuffers​ 这条函数的入口在哪里？”

  • 原生做法（Windows 示例）

    typedef void (*GL_GENBUFFERS)(GLsizei, GLuint*);
GL_GENBUFFERS glGenBuffers = (GL_GENBUFFERS)wglGetProcAddress("glGenBuffers");
// 之后就能正常调用 glGenBuffers(1, &buffer);

    对每个函数都要如此重复，既繁琐又易错

  • 使用加载库的好处

    • 自动化：指定所需的 OpenGL 版本和扩展后，工具帮你生成对应的加载代码
    • 精简：只包含用到的函数，减少冗余
    • 跨平台：隐藏各系统查址接口的差异，如 Windows 的 wglGetProcAddress​、Linux 的 glXGetProcAddress​、macOS 的 NSGL​

创建流程

GLAD

  1. 首先，新建一个.cpp​文件，包含GLAD和GLFW头文件

    #include <glad/glad.h>
#include <GLFW/glfw3.h>

    请确认是在包含GLFW的头文件之前包含了GLAD的头文件。GLAD的头文件包含了正确的OpenGL头文件，所以需要在其它依赖于OpenGL的头文件之前包含GLAD

  2. 创建main函数，实例化GLFW窗口

    int main()
{
    glfwInit();
    glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 3);
    glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 3);
    glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_CORE_PROFILE);
    //for Mac:
    //glfwWindowHint(GLFW_OPENGL_FORWARD_COMPAT, GL_TRUE);

    return 0;
}

    1. 在main函数中调用glfwInit函数来初始化GLFW

    2. 使用glfwWindowHint函数来配置GLFW，将主版本号(Major)和次版本号(Minor)都设为3

      glfwWindowHint函数的第一个参数代表选项的名称，可以从很多以GLFW_开头的枚举值中选择；第二个参数接受一个整型，用来设置这个选项的值

    3. 告诉GLFW我们使用的是核心模式(Core-profile)，使用核心模式意味着只能使用OpenGL功能的一个子集

      在 OpenGL 中，“核心模式”（Core Profile）和“兼容模式”（Compatibility Profile）其实是对同一个 API 的两种不同打包方式，使用核心模式，能够确保你的程序只使用现代、可持续发展的 OpenGL 功能，避免踩到已废弃接口的坑，提高可维护性和跨平台一致性

  3. 创建窗口对象，这个窗口对象存放了所有和窗口相关的数据

    GLFWwindow* window = glfwCreateWindow(800, 600, "LearnOpenGL", NULL, NULL);
if (window == NULL)
{
    std::cout << "Failed to create GLFW window" << std::endl;
    glfwTerminate();
    return -1;
}
glfwMakeContextCurrent(window);

    glfwCreateWindow函数前两个参数为窗口的宽和高，第三个参数表示这个窗口的名称（标题），最后两个参数暂时忽略。这个函数将会返回一个GLFWwindow对象，会在其它的GLFW操作中多次使用到

    创建完窗口后，把 window​ 这个窗口背后的 OpenGL 上下文，绑定（“make current”）到当前的线程上

  4. 初始化GLAD，即可调用OpenGL函数

    if (!gladLoadGLLoader((GLADloadproc)glfwGetProcAddress))
{
    std::cout << "Failed to initialize GLAD" << std::endl;
    return -1;
}

    向GLAD传入了用来加载系统相关的OpenGL函数指针地址的函数。GLFW给我们的是glfwGetProcAddress​，它根据我们编译的系统定义了正确的函数

视口

  1. 创建一个回调函数(Callback Function)，用于当用户改变窗口的大小的时候，调整视口

    void framebuffer_size_callback(GLFWwindow* window, int width, int height)
{
    glViewport(0, 0, width, height);
}

    这个函数需要一个GLFWwindow作为它的第一个参数，以及两个整数表示窗口的新维度

  2. 在函数中调用glViewport函数来设置视口的尺寸（如上）

    glViewport(0, 0, 800, 600);

    glViewport函数前两个参数控制窗口左下角的位置，第三个和第四个参数控制渲染窗口的宽度和高度

  3. 调用这个函数，通知GLFW每当窗口调整大小的时候调用这个函数

    glfwSetFramebufferSizeCallback(window, framebuffer_size_callback);

引擎

  1. 添加渲染循环（Render Loop）

    while(!glfwWindowShouldClose(window))
{
    glfwSwapBuffers(window);
    glfwPollEvents();    
}
    • glfwWindowShouldClose函数在每次循环的开始前检查一次GLFW是否被要求退出，如果是的话，该函数返回true​，渲染循环将停止运行，之后就可以关闭应用程序
    • glfwPollEvents函数检查有没有触发什么事件（比如键盘输入、鼠标移动等）、更新窗口状态，并调用对应的回调函数（可以通过回调方法手动设置）
    • glfwSwapBuffers函数会交换颜色缓冲（它是一个储存着GLFW窗口每一个像素颜色值的大缓冲），它在这一迭代中被用来绘制，并且将会作为输出显示在屏幕上

  2. 调用glfwTerminate，在渲染循环结束后释放之前分配的所有资源

    glfwTerminate();
return 0;

输入

  1. 创建一个processInput函数装载GLFW的glfwGetKey函数

    void processInput(GLFWwindow *window)
{
    if(glfwGetKey(window, GLFW_KEY_ESCAPE) == GLFW_PRESS)
        glfwSetWindowShouldClose(window, true);
}

    glfwGetKey函数需要一个窗口以及一个按键作为输入，这个函数将会返回这个按键是否正在被按下

  2. 在渲染循环的每一个迭代中调用processInput

    while (!glfwWindowShouldClose(window))
{
    processInput(window);

    glfwSwapBuffers(window);
    glfwPollEvents();
}

渲染

所有的渲染(Rendering)操作都应放到渲染循环中，代码将会是这样的：

// 渲染循环
while(!glfwWindowShouldClose(window))
{
    // 输入
    processInput(window);

    // 渲染指令
    ...

    // 检查并调用事件，交换缓冲
    glfwPollEvents();
    glfwSwapBuffers(window);
}

在每个新的渲染迭代开始要进行清屏处理，否则仍能看见上一次的渲染结果。调用glClear函数来清空屏幕的颜色缓冲，它接受一个缓冲位(Buffer Bit)来指定要清空的缓冲，可能的缓冲位有GL_COLOR_BUFFER_BIT，GL_DEPTH_BUFFER_BIT和GL_STENCIL_BUFFER_BIT

glClearColor(0.2f, 0.3f, 0.3f, 1.0f);
glClear(GL_COLOR_BUFFER_BIT);

同时调用了glClearColor来设置清空屏幕所用的颜色，当调用glClear函数，清除颜色缓冲之后，整个颜色缓冲都会被填充为glClearColor里所设置的颜色

glClearColor函数是一个状态设置函数，而glClear函数则是一个状态使用的函数，它使用了当前的状态来获取应该清除为的颜色
